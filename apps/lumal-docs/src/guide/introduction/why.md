# 为什么选择 Lumal

## 轻量、边界清晰

很多 Admin 模板把业务字段、状态码和 UI 框架深度耦合。Lumal 强制：

- **框架包不承载业务协议**：菜单 / 分页 / 会话使用稳定语义字段，非标准后端在应用层适配。
- **可选能力独立成包**：图表、DataV、驾驶舱、兼容层互不绑架默认安装体积。
- **Element Plus 为 peer**：不锁死你的 UI 版本策略，也不把 UI 库打进 core 依赖。

## 适合的场景

- 中小型中后台，希望快速有布局、权限、CRUD 基线
- 需要大屏 / 驾驶舱，但不想把地图与图表引擎塞进 core
- 从 Vben 类项目迁移，需要兼容层过渡
- monorepo 内同时维护多个示例与可发布包

## 不适合的场景

- 需要完整复刻 Vben 全量 API / 多 UI 库一键切换（请直接使用对应生态）
- 需要框架内置 i18n / VXE 默认依赖（core 默认不引入，应用自行组合）

## 与示例的关系

| 应用 | 角色 |
| --- | --- |
| `lumal-admin` | 公开 API 验收与系统管理示例 |
| `lumal-cockpit` | 驾驶舱编排示例 |
| `lumal-datav-guide` | DataV 组件可视文档 |
| `create-lumal-admin` | 外部项目起盘，不依赖 monorepo 源码路径 |
